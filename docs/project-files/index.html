Project Files Naming Guide

Optimize document organization and team collaboration by standardizing your creative and technical project files.

Keeping design files, codebases, assets, and project logs consistent is essential for fast retrieval. The built-in Project Files template provides a reliable structure for organizing corporate, freelance, and personal files.

Default Template Structure

The default Project Files template uses an underscore separator (_) and consists of the following components:

  • Date Created: Date format in YYYYMMDD (e.g. 20260709) to keep files sorted chronologically in your filesystem.
  • Project Code: A short code representing the client or project name (e.g. PRJ101).
  • Description: A brief description of the specific asset or content (e.g. PromoShot).
  • Index: A two-digit auto-incrementing counter (e.g. 01, 02) to manage versioning and sequential asset lists.
  • Extension: Preserves the original file extension automatically.

Example output filename: 20260709_PRJ101_PromoShot_01.png

Best Practices for Customizing

Every project team has distinct operational needs. When tailoring this template in the Field Builder, consider these adjustments:

  • Differentiate Drafts: Add a custom dropdown component for version stages, such as draft, review, and final.
  • Incorporate User Initials: Add a text field for creator initials (e.g. RM) to identify who worked on a specific file.
  • Enforce Snake or Kebab Case: Use the global casing controls at the bottom of the template builder to automatically force lowercase or camelCase filenames depending on cross-platform compatibility needs.
