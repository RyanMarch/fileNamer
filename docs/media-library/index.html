Media Library Naming Guide

Ensure seamless scanning and metadata scraping in home media servers like Plex, Radarr, or Emby.

Media servers require highly structured file naming conventions to correctly recognize movie and TV show titles, years, and file resolutions. The built-in Media Library template provides an optimized standard structure to avoid unrecognized media.

Default Template Structure

The default Media Library template uses a space separator ( ) and contains the following fields:

  • Movie Title: The official name of the film (e.g. Avatar).
  • Release Year: Four-digit release year in parentheses or brackets (e.g. 2009).
  • IMDb ID: The official IMDb tag (e.g. tt0499549) to guarantee metadata matching.
  • Resolution: The quality parameter (e.g. 1080p or 2160p).
  • Source: The origin of the file (e.g. BluRay or WEB-DL).
  • Video Codec: The visual encoder format (e.g. x264 or x265).
  • Audio Codec: The sound format (e.g. DTS-HD or TrueHD).
  • Release Group: The encoder tag (e.g. RARBG).
  • Extension: Preserves the media file's original suffix (e.g. .mkv).

Example output filename: Avatar 2009 tt0499549 1080p BluRay x264 DTS-HD RARBG.mkv

Best Practices for Customizing

Home theater environments differ in organization. Try customizing this layout in the Template Builder:

  • Plex Style Bracket IDs: Wrap the IMDb ID or TMDB ID field in braces (e.g., {imdb-tt0499549}) in the builder to align with the latest Plex Agent scanner guidelines.
  • TV Show Support: Replace the Movie Title with fields for Show Title, Season (S01), and Episode (E01) to standardize entire television series.
  • Directory Structures: Use slashes in your separator or static components if you want to preview nested folder generation layouts (e.g., Show Name/Season 01/Show Name - S01E01.mkv).
